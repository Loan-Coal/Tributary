## Objective

Write a minimal seed script that loads all golden scenario JSON files into a running Neo4j 5 instance. This is NOT the full Wave 2 ingestion pipeline (that comes from the graph colleague). It is a dev-utility seed that creates exactly the nodes and relationships needed to verify the graph is traversable: Entity, Account, Transaction, Counterparty, PresenceRecord, PriorPeriodLoss, and the ownership/holds/records/presence/loss relationship edges.

The exit gate for Wave 1 is: `make ingest` runs without error; Neo4j Browser shows the Meridian Group entities, ownership chain, and the 185-day presence record.

## Implementation notes

### Config
Import from `src/tributary/config/settings.py`:
- `NEO4J_URI`, `NEO4J_USER`, `NEO4J_PASSWORD`
- `DATA_DIR` (defaults to `"data"`)

### Data loading
Load golden files from `{DATA_DIR}/golden/`:
- `entities.json` → validate as list of `EntityRecord`
- `accounts.json` → validate as list of `AccountRecord`
- `ownership.json` → validate as list of `OwnershipRecord`
- `transactions.json` → validate as list of `TransactionRecord`
- `presence_records.json` → validate as list of `PresenceRecord`
- `prior_losses.json` → validate as list of `PriorPeriodLoss`

Use `from tributary.common import EntityRecord, AccountRecord, ...` for validation (fail fast if JSON doesn't parse).

### Cypher patterns (all MERGE for idempotency)

```cypher
-- Entity
MERGE (e:Entity {entity_id: $entity_id})
SET e.name = $name,
    e.entity_type = $entity_type,
    e.incorporation_jurisdiction = $incorporation_jurisdiction,
    e.resident_jurisdiction = $resident_jurisdiction,
    e.is_group_member = $is_group_member

-- Ownership
MATCH (o:Entity {entity_id: $owner_id})
MATCH (s:Entity {entity_id: $owned_id})
MERGE (o)-[:OWNS {pct: $pct, effective_from: date($eff_from)}]->(s)

-- Account
MERGE (a:Account {account_id: $account_id})
SET a += {entity_id: $entity_id, account_name: $name, account_type: $type}
WITH a
MATCH (e:Entity {entity_id: $entity_id})
MERGE (e)-[:HOLDS]->(a)

-- Transaction
MERGE (t:Transaction {transaction_id: $transaction_id})
SET t += {all fields...}
WITH t
MATCH (a:Account {entity_id: $source_entity_id})  -- use entity's account
MERGE (a)-[:RECORDS]->(t)

-- PresenceRecord
MERGE (p:PresenceRecord {presence_id: $presence_id})
SET p += {all fields...}
WITH p
MATCH (e:Entity {entity_id: $entity_id})
MERGE (e)-[:HAS_PRESENCE]->(p)

-- PriorPeriodLoss
MERGE (l:PriorPeriodLoss {loss_id: $loss_id})
SET l += {all fields...}
WITH l
MATCH (e:Entity {entity_id: $entity_id})
MERGE (e)-[:HAS_PRIOR_LOSS]->(l)
```

### Transaction node fields
Store all TransactionRecord fields. For Decimal fields, store as float (Neo4j doesn't have Decimal type — note this in DECISIONS.md or ISSUES.md). Dates as Neo4j `date()` types.

### Error handling
- If Neo4j is not reachable: print clear error and exit 1 with message "Start Neo4j with `docker-compose up -d` before seeding"
- If a JSON file is missing: `IngestionError` with filename
- If validation fails: `IngestionError` with which record failed

### Integration test
`tests/integration/test_seed.py`:
- Attempt to connect to Neo4j; if not available, `pytest.skip("Neo4j not running")`
- If available: run seed, then query node counts
- Assert: 3 Entity nodes, 2 OWNS relationships, 9 Transaction nodes, 1 PresenceRecord, 1 PriorPeriodLoss
